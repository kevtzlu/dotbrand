## 簡介

### 目的

GC Double Check Protocol 定義了如何與總承包商進行估算驗證，確保：

1. **透明度** - GC 理解估算的邏輯和假設
2. **準確度** - GC 提供反饋以改進估算
3. **認同度** - GC 同意最終估算結果
4. **可追溯性** - 所有決策都有記錄

### 核心原則

- ✅ **透明**: 完全解釋估算邏輯
- ✅ **協作**: 與 GC 進行真誠的對話
- ✅ **靈活**: 根據 GC 反饋進行調整
- ✅ **可記錄**: 所有反饋都有記錄

## Double Check 流程

### Step 1: 準備 Double Check 會議

**時間**: 在生成初步估算後立即進行

**參與者**:
- [ ] 估算師 (Estimator)
- [ ] GC 項目經理 (Project Manager)
- [ ] GC 成本工程師 (Cost Engineer)
- [ ] 其他相關人員 (如需要)

**準備材料**:
- [ ] 新項目特性檔案 (YAML)
- [ ] 相似度計算結果
- [ ] 參考案例信息
- [ ] 調整因子詳解
- [ ] 初步估算結果
- [ ] Sanity Check 報告
- [ ] GC Double Check 清單

**會議時長**: 1-2 小時 (根據置信度等級)

### Step 2: 呈現估算邏輯

**目標**: 幫助 GC 理解估算的邏輯和假設

**呈現內容**:

#### **2.1 案例庫方法簡介**

```
"我們使用案例庫方法進行估算。這個方法基於以下邏輯：

1. 從案例庫中找到最相似的完成項目
2. 使用該項目的成本/SF 作為基準
3. 根據新項目與參考案例的差異進行調整
4. 生成新項目的成本估算

這個方法的優點是：
- 基於實際完成項目的數據
- 考慮了項目特定的因素
- 結果更加可靠和可追溯
"
```

#### **2.2 參考案例信息**

呈現以下信息：

```
參考案例: Colton Hospital II - Behavioral Health Center
  - 建築面積: 35,062 SF
  - 工期: 20 個月
  - 完成日期: 2025-12-31
  - 最終成本: $8,862,724
  - 成本/SF: $253
  - 相似度評分: 85.75%
  - 置信度等級: HIGH
```

#### **2.3 相似度評分詳解**

呈現 6 個維度的相似度：

```
相似度評分分解:
  1. 建築類型匹配: 85% (同類型 Healthcare)
  2. 地區相似度: 80% (Regional Multiplier 差異)
  3. 規模相似度: 85% (建築面積相差 42%)
  4. 工期相似度: 95% (工期差異 2 個月)
  5. 改造等級匹配: 85% (改造等級相差 1)
  6. 複雜度相似度: 95% (複雜度評分差異 2 分)

總體相似度: 85.75% → 置信度等級: HIGH
```

#### **2.4 調整因子詳解**

逐個解釋每個調整因子：

```
調整因子應用:

Step 1: 建築類型調整 (Healthcare MOB vs Behavioral Health)
  因子: 0.95x
  理由: MOB 的 MEP 複雜度略低於 Behavioral Health
  
Step 2: 地區調整 (CA Coastal vs CA Inland)
  因子: 1.09x
  理由: Regional Multiplier 差異 (1.25 vs 1.15)
  
Step 3: 規模調整 (50,000 SF vs 35,062 SF)
  因子: 0.99x
  理由: 規模經濟效應 (更大的項目成本/SF 略低)
  
Step 4: 工期調整 (18 個月 vs 20 個月)
  因子: 0.98x
  理由: 工期更短，間接成本更低
  
Step 5: 改造強度調整 (Level 2 vs Level 1)
  因子: 1.35x
  理由: MEP 更換比例更高 (50% vs 75%)
  
Step 6: 複雜度調整 (Score 11 vs Score 9)
  因子: 1.10x
  理由: 複雜度評分更高，風險更大

淨調整因子: 0.95 × 1.09 × 0.99 × 0.98 × 1.35 × 1.10 = 1.49x
```

#### **2.5 最終估算結果 (updated v1.1)**

呈現最終成本估算：

```
基準成本/SF (Colton): $253
淨調整因子: 1.49x
調整後成本/SF: $253 × 1.49 = $377/SF

建築面積: 50,000 SF
總成本估算: $377 × 50,000 = $18,850,000

成本分解:
  - Hard Costs (含 Div 01): $14,514,500 (77%)
  - Soft Costs (Fee/Contingency): $4,335,500 (23%)
  
總成本: $18,850,000
```

### Step 5: 根據反饋進行調整

**目標**: 根據 GC 反饋進行必要的調整

**調整流程**:

#### **5.1 評估反饋的合理性**

對每個反饋進行評估：

```
反饋 1: 建築類型調整應該是 0.90x 而不是 0.95x

評估:
  - GC 的理由是什麼？ _______________________________________________
  - 這個理由是否合理？ [ ] 是 [ ] 否
  - 這會對估算產生什麼影響？ ________________________________________
  - 我們是否應該接受這個反饋？ [ ] 是 [ ] 否
```

#### **5.2 進行調整**

如果決定接受反饋，進行調整：

```
原始調整因子: 0.95x
GC 建議: 0.90x
決定: 接受 GC 建議

新的淨調整因子: 0.90 × 1.09 × 0.99 × 0.98 × 1.35 × 1.10 = 1.41x
新的調整後成本/SF: $253 × 1.41 = $357/SF
新的總成本: $357 × 50,000 = $17,850,000
```

#### **5.3 生成修訂的估算**

重新計算所有成本分類：

```
新的成本分解:
  - 直接成本: $8,925,000 (50%)
  - 間接成本: $3,800,250 (21.3%)
  - 軟成本: $3,034,500 (17%)
  - 利潤/費用: $1,071,000 (6%)
  - 應急預留: $1,019,250 (5.8%)
  
新的總成本: $17,850,000
```

## 反饋處理和調整

### 反饋分類

**Type A: 同意**
- 無需調整
- 記錄為「已批准」

**Type B: 部分同意**
- 需要進行微調
- 根據 GC 建議進行調整
- 重新計算估算

**Type C: 不同意**
- 需要進行詳細討論
- 可能需要進行現場評估
- 可能需要使用替代方法

### 調整流程

```
收集 GC 反饋
  ↓
分類反饋 (A/B/C)
  ↓
  ├─→ Type A: 記錄為已批准
  │
  ├─→ Type B: 進行微調
  │   ├─→ 更新相關參數
  │   ├─→ 重新計算估算
  │   └─→ 呈現修訂結果
  │
```
