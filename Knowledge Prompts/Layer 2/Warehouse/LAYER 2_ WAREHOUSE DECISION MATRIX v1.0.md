## 簡介

### 目的

Warehouse Decision Matrix 提供了一個結構化的工具，用於快速評估倉庫項目的特性，並根據這些特性進行初步的成本估算和決策。

### 使用時機

- **Stage 1 (初始驗證)**: 在確認項目類型和基本特性時使用
- **Stage 3 (方法與初步估算)**: 在確定施工方法和複雜度時使用
- **快速估算**: 當需要快速提供初步成本估算時使用

### 目標準確度

- **快速估算**: ±15-20% (基於決策矩陣)
- **詳細估算**: ±5-10% (基於 LAYER1 完整計算)

## 決策樹

### 倉庫項目分類決策樹

```
開始
  ↓
[Q2: 淨空高度]
  ├─ < 24 ft → 標準倉庫 (路徑 A)
  │   ├─ [Q3: 辦公比例]
  │   │   ├─ < 10% → 純倉庫 (成本: $75-105/SF)
  │   │   └─ > 10% → 混合倉庫 (成本: $130-180/SF)
  │   └─ 進行詳細估算
  │
  ├─ 24-32 ft → 中等高度 (路徑 B)
  │   ├─ [Q8: 特殊系統]
  │   │   ├─ 無特殊系統 → 標準中等高度 (成本: $130-160/SF)
  │   │   └─ 包含 ESFR → 高架倉庫 (成本: $150-200/SF)
  │   └─ 進行詳細估算
  │
  └─ > 32 ft → 高架倉庫 (路徑 C)
      ├─ [Q8: 特殊系統]
      │   ├─ 無特殊系統 → 基本高架 (成本: $200-250/SF)
      │   ├─ 包含 ESFR → ESFR 高架 (成本: $220-280/SF)
      │   └─ 包含多個系統 → 複雜高架 (成本: $250-350/SF)
      └─ 進行詳細估算
```

## 成本計算器

### 快速成本計算公式

```
初步成本估算 = 基礎成本 × 地區乘數 × 勞動力乘數 × 工期乘數 × (1 + 特殊系統溢價) × (1 + 地質調整) × (1 + 軟成本%)
```

#### 例子 1: 標準倉庫 (Inland Empire)

## 使用指南

### 何時使用決策矩陣

✅ **使用決策矩陣當**:
- 需要快速初步估算 (±15-20%)
- 項目處於概念階段
- 需要快速評估項目可行性
- 需要與多個選項進行比較

❌ **不使用決策矩陣當**:
- 需要高精度估算 (±5-10%)
- 項目已進入設計階段
- 需要詳細的 BOQ 和成本分解
- 需要與承包商進行談判

### 決策矩陣到詳細估算的過渡

1. **使用決策矩陣進行快速評估** (準確度 ±15-20%)
2. **收集更多詳細信息** (地質報告、設計圖紙等)
3. **使用 LAYER1 進行詳細計算** (準確度 ±5-10%)
4. **進行健全性檢查** (與案例數據庫進行比較)
